# 智能小车模型训练说明

本目录是模型训练和模型转换的最小工程：

- `lanenet/`：车道线 LaneNet 训练、评估、ONNX 导出。
- `yolov5/`：地标/交通标志 YOLOv5 训练、验证、ONNX 导出。
- `scripts/`：YOLO 数据划分和 ATC 转 OM 脚本。

训练建议在带 NVIDIA GPU 的 Linux、WSL2 Ubuntu、服务器或云主机上完成，不建议在 Atlas 小车开发板上训练。训练完成后导出 ONNX，再使用 CANN ATC 转成小车端可加载的 `.om`。

## 1. 提交范围

仓库只保留训练所需源码和说明文档，不提交数据集、训练输出和模型权重。以下内容需要你在本地自行准备或由脚本生成：

```text
datasets_final/                         # 标注好的数据集，通常放在 Car22 同级目录
car_model_training/lanenet/Datasets/    # LaneNet 转换后的训练数据
car_model_training/lanenet/runs/        # LaneNet 训练和评估输出
car_model_training/lanenet/checkpoints/ # 可选的学长预训练权重
car_model_training/yolov5/runs/         # YOLOv5 训练输出
car_model_training/deploy_models/       # ATC 转出的 OM
*.pt / *.model / *.onnx / *.om / *.whl  # 二进制权重和安装包
```

如果需要基于学长 LaneNet 权重继续训练，把文件手动放到：

```text
car_model_training/lanenet/checkpoints/lanenet_epoch_38_batch_8_AUG.model
```

如果 YOLOv5 训练要使用 `yolov5s.pt`，可以让 YOLOv5 自动下载，也可以手动放到：

```text
car_model_training/yolov5/yolov5s.pt
```

## 2. 推荐环境

LaneNet 依赖 `torch_scatter`，它和 Python、PyTorch、CUDA 版本强绑定。当前建议环境：

- Linux x86_64 或 WSL2 Ubuntu
- Python 3.8
- CUDA 11.6
- PyTorch 1.13.x + cu116
- torchvision 0.14.x + cu116

创建环境：

```bash
conda create -n car-train python=3.8 -y
conda activate car-train
```

安装 PyTorch：

```bash
pip install torch==1.13.1+cu116 torchvision==0.14.1+cu116 --extra-index-url https://download.pytorch.org/whl/cu116
```

安装 LaneNet 依赖：

```bash
cd car_model_training/lanenet
pip install -r requirements.txt
pip install "numpy==1.24.4" tqdm matplotlib jupyter
pip install torch-scatter==2.1.1 -f https://data.pyg.org/whl/torch-1.13.1+cu116.html
```

如果你已经有匹配当前环境的 `torch_scatter` 本地 wheel，也可以改用：

```bash
pip install ./torch_scatter-xxx.whl
```

安装 YOLOv5 依赖：

```bash
cd ../yolov5
pip install -r requirements.txt
```

检查 GPU：

```bash
python -c "import torch; print(torch.__version__); print(torch.version.cuda); print(torch.cuda.is_available())"
```

输出 `True` 才表示 PyTorch 能使用 CUDA。

## 3. LaneNet 数据准备

车道线原始数据应为 LabelMe 格式：

```text
datasets_final/lane/images
datasets_final/lane/jsons
```

进入 LaneNet 目录：

```bash
cd car_model_training/lanenet
```

转换数据：

```bash
python data_prosses.py \
  --input-root "../../datasets_final/lane" \
  --output-root "Datasets/Tusimple" \
  --img-width 1280 \
  --img-height 720 \
  --mask-width 16 \
  --seed 2026
```

脚本会生成：

```text
Datasets/Tusimple/labels_tusimple.json
Datasets/Tusimple/training/gt_image
Datasets/Tusimple/training/gt_binary_image
Datasets/Tusimple/training/gt_instance_image
Datasets/Tusimple/txt_for_local/train.txt
Datasets/Tusimple/txt_for_local/val.txt
Datasets/Tusimple/txt_for_local/test.txt
```

如果只想先确认转换效果，可以加 `--debug`：

```bash
python data_prosses.py \
  --input-root "../../datasets_final/lane" \
  --output-root "Datasets/Tusimple_debug" \
  --img-width 1280 \
  --img-height 720 \
  --debug
```

## 4. LaneNet 训练

基于学长已有权重继续训练：

```bash
python train_lanenet.py \
  --txt-root "Datasets/Tusimple/txt_for_local" \
  --output-dir "runs/lanenet_output" \
  --model-module model3 \
  --pretrained "checkpoints/lanenet_epoch_38_batch_8_AUG.model" \
  --epochs 50 \
  --batch-size 8 \
  --device cuda \
  --run-name lane_custom
```

这相当于在学长模型基础上微调。数据集和当前赛道更接近时，一般比从零训练更稳。

如果从头训练，去掉 `--pretrained`：

```bash
python train_lanenet.py \
  --txt-root "Datasets/Tusimple/txt_for_local" \
  --output-dir "runs/lanenet_output" \
  --model-module model3 \
  --epochs 50 \
  --batch-size 8 \
  --device cuda \
  --run-name lane_custom_from_scratch
```

显存不够时，把 `--batch-size` 改成 `4` 或 `2`。

## 5. LaneNet 评估

```bash
python evaluate_lanenet.py \
  --weights "runs/lanenet_output/lane_custom_final_batch_8.model" \
  --txt-root "Datasets/Tusimple/txt_for_local" \
  --split test \
  --model-module model3 \
  --device cuda \
  --output-dir "runs/lanenet_eval" \
  --save-vis \
  --vis-count 50
```

输出：

```text
runs/lanenet_eval/metrics.json
runs/lanenet_eval/metrics.csv
runs/lanenet_eval/vis/
```

重点看：

```text
foreground_iou       车道线像素 IoU
foreground_precision 预测为车道线的像素里有多少是真的
foreground_recall    真实车道线像素里有多少被找出来
foreground_f1        precision 和 recall 的综合指标
```

可视化图中绿色是真实标注，红色是模型预测，黄色是预测和标注重合的位置。最终仍建议打开 `vis/` 看预测线是否落在当前车道线上。

## 6. YOLO 数据准备

YOLO 数据应整理为：

```text
datasets_final/yolo/images
datasets_final/yolo/labels
datasets_final/yolo/classes.txt
```

在 `car_model_training` 目录下执行：

```bash
python scripts/prepare_yolo_dataset.py \
  --dataset-root "../datasets_final/yolo" \
  --train-ratio 0.8 \
  --val-ratio 0.1 \
  --seed 2026
```

脚本会生成或更新：

```text
datasets_final/yolo/data.yaml
datasets_final/yolo/train.txt
datasets_final/yolo/val.txt
datasets_final/yolo/test.txt
```

当前类别顺序：

```text
0 back
1 crosswalk
2 left
3 park
4 right
```

类别顺序必须和小车端 `base/src/models/yolov5.py` 及 `ros2` 中的 YOLO 封装保持一致。

## 7. YOLOv5 训练和验证

进入 YOLOv5 目录：

```bash
cd car_model_training/yolov5
```

训练：

```bash
python train.py \
  --img 640 \
  --batch-size 8 \
  --epochs 100 \
  --data "../../datasets_final/yolo/data.yaml" \
  --weights yolov5s.pt \
  --project runs/train \
  --name car_yolo
```

验证：

```bash
python val.py \
  --weights runs/train/car_yolo/weights/best.pt \
  --data "../../datasets_final/yolo/data.yaml" \
  --img 640
```

如果显存不够，调小 `--batch-size`。

## 8. 导出 ONNX

小车端最终加载 `.om`，完整链路是：

```text
LaneNet: .model -> .onnx -> .om
YOLOv5 : .pt    -> .onnx -> .om
```

导出 LaneNet ONNX：

```bash
cd car_model_training/lanenet
python export_lanenet_onnx.py \
  --weights "runs/lanenet_output/lane_custom_final_batch_8.model" \
  --output "runs/lanenet_output/lane_custom_final_batch_8.onnx" \
  --model-module model3 \
  --device cpu \
  --batch-size 1 \
  --img-height 256 \
  --img-width 512 \
  --opset 11
```

LaneNet 输入尺寸必须和小车端保持一致：

```text
images:1,3,256,512
```

导出 YOLOv5 ONNX：

```bash
cd car_model_training/yolov5
python export.py \
  --weights runs/train/car_yolo/weights/best.pt \
  --include onnx \
  --opset 11 \
  --img 640 \
  --batch-size 1 \
  --device cpu
```

YOLOv5 输入尺寸：

```text
images:1,3,640,640
```

## 9. ATC 转 OM

ATC 是 CANN 工具链里的模型转换工具，需要在安装了 Ascend CANN 的 Linux/Atlas 环境里执行。

进入训练工程根目录：

```bash
cd car_model_training
```

加载 CANN 环境。默认安装路径通常是：

```bash
export CANN_INSTALL_PATH="/usr/local/Ascend"
source "${CANN_INSTALL_PATH}/ascend-toolkit/set_env.sh"
```

如果 `atc` 报找不到 `libascend_hal.so`，补充 devlib：

```bash
export LD_LIBRARY_PATH="$(find "${CANN_INSTALL_PATH}/ascend-toolkit" -path "*/x86_64-linux/devlib" -type d 2>/dev/null | head -n 1):${LD_LIBRARY_PATH}"
```

检查 ATC：

```bash
which atc
atc --help | head -n 20
```

有些 CANN 版本不支持 `atc --version`，`atc --help` 能输出即可。

Atlas 200I DK A2 常用 `Ascend310B4`。执行转换：

```bash
export TE_PARALLEL_COMPILER=1
CANN_INSTALL_PATH="/usr/local/Ascend" \
SOC_VERSION=Ascend310B4 \
bash scripts/atc_convert_models.sh
```

默认输入：

```text
lanenet/runs/lanenet_output/lane_custom_final_batch_8.onnx
yolov5/runs/train/car_yolo/weights/best.onnx
```

默认输出：

```text
deploy_models/lanenet.om
deploy_models/yolo.om
```

如果路径不同，用环境变量覆盖：

```bash
LANENET_ONNX="lanenet/runs/lanenet_output/your_lanenet.onnx" \
YOLO_ONNX="yolov5/runs/train/car_yolo/weights/your_yolo.onnx" \
OUTPUT_DIR="deploy_models" \
CANN_INSTALL_PATH="/usr/local/Ascend" \
SOC_VERSION=Ascend310B4 \
bash scripts/atc_convert_models.sh
```

如果只转 LaneNet，可以给 YOLO 一个不存在的占位路径：

```bash
LANENET_ONNX="senior_models/onnx/lanenet_my_4_batch_8.onnx" \
YOLO_ONNX="__skip_yolo__.onnx" \
OUTPUT_DIR="senior_models/om/lanenet_my_4" \
CANN_INSTALL_PATH="/usr/local/Ascend" \
SOC_VERSION=Ascend310B4 \
bash scripts/atc_convert_models.sh
```

脚本会跳过不存在的输入模型。

转换完成后复制到小车运行目录：

```bash
cp deploy_models/lanenet.om ../base/weights/lanenet.om
cp deploy_models/yolo.om ../base/weights/yolo.om
cp deploy_models/lanenet.om ../ros2/ros2_ws/src/smart_car_nodes/weights/lanenet.om
cp deploy_models/yolo.om ../ros2/ros2_ws/src/smart_car_nodes/weights/yolo.om
```

## 10. 常见问题

`torch_scatter` 安装失败：

确认 Python、PyTorch、CUDA 是否匹配。当前推荐 Linux x86_64 + Python 3.8 + PyTorch 1.13 + CUDA 11.6。

`numpy.core.multiarray failed to import`：

通常是 NumPy 2.x 与 pandas、opencv 等二进制包不兼容：

```bash
pip install "numpy==1.24.4"
```

LaneNet 训练报显存不足：

减小 `--batch-size`。

LaneNet 训练时报 `cv2.resize` 的 `!ssize.empty()`：

通常是 `txt_for_local` 里的图片路径读不到。检查 `--txt-root` 是否指向 `Datasets/Tusimple/txt_for_local`，再打开 `train.txt` 看三列路径是否真实存在。

YOLO 类别数量或名称不对：

确认使用的是 `../../datasets_final/yolo/data.yaml`，并检查小车端 `base/src/models/yolov5.py` 中的类别顺序。

YOLO 报 `ModuleNotFoundError: No module named 'comet_ml'`：

`comet_ml` 只是可选日志工具。当前训练脚本不需要它；如果仍报错，确认使用的是本目录下当前版本的 `yolov5/train.py`。

ATC 提示 `No module named 'sympy'`：

```bash
python -m pip install sympy
```

ATC 提示算子编译失败或 `Broken pipe`：

先限制并行编译，降低 WSL 下的转换压力：

```bash
export TE_PARALLEL_COMPILER=1
CANN_INSTALL_PATH="/usr/local/Ascend" \
SOC_VERSION=Ascend310B4 \
bash scripts/atc_convert_models.sh
```

如果仍失败，查看最近日志：

```bash
find "$HOME/ascend/log" -type f -mmin -30 2>/dev/null | sort
grep -R "ERROR\\|TraceBack\\|Compile operator failed" "$HOME/ascend/log" -n 2>/dev/null | tail -n 120
```

ATC 提示找不到输入名 `images`：

查看 ONNX 输入名：

```bash
python -c "import onnx; m=onnx.load('lanenet/runs/lanenet_output/lane_custom_final_batch_8.onnx'); print([i.name for i in m.graph.input])"
```

如果不是 `images`，把 `scripts/atc_convert_models.sh` 中对应的 `--input_shape` 输入名改成实际名称。
