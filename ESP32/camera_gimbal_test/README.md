# MG90S 摄像头云台调试

这个目录是两个 MG90S 舵机的独立调试工程。它使用当前小车已经预留的两个舵机信号引脚：`GPIO25` 和 `GPIO26`，不会覆盖电机 PWM 或 PID 实验工程。

## 接线

MG90S 通常有三根线：

- 红色：电源正极。
- 棕色或黑色：地线。
- 橙色、黄色或白色：PWM 信号线。

线色可能随厂商变化，应以舵机插头旁的 `VCC`、`GND`、`SIG` 标识为准。

| 云台轴 | 舵机信号线 | 舵机红线 | 舵机棕/黑线 |
| --- | --- | --- | --- |
| 水平左右 Pan | ESP32 `GPIO25` | 外部稳压 5V 正极 | 外部 5V GND |
| 竖直上下 Tilt | ESP32 `GPIO26` | 外部稳压 5V 正极 | 外部 5V GND |

还必须增加一根公共地线：

```text
外部 5V 电源 GND <----> ESP32 开发板 GND
```

两个 MG90S 建议使用独立的稳定 5V 电源，额定输出至少 2A，推荐 3A。不要把舵机红线接到 ESP32 的 `3.3V`，也不要只依靠 USB 给两路舵机供电。可以在两个舵机附近的外部 5V 与 GND 间并联一个 470 至 1000 uF 电解电容，正极接 5V、负极接 GND，以减轻动作瞬间的压降。

## 上传

1. 用 Arduino IDE 打开 `camera_gimbal_test.ino`。
2. 选择当前 ESP32 IOT & Robot Board 使用的开发板和串口。
3. 编译并上传；串口监视器设置为 `115200` 波特率和 `Newline` 行结束符。
4. 上电后应看到 `INFO,camera_gimbal_test ready`，两个舵机会被命令到代码中配置的默认中位。

工程本地包含 `ESP32_Servo.cpp/.h`，不需要额外从 Arduino 库管理器安装舵机库。

## 安装与标定

1. 初次上电先不要把舵盘锁死在机械支架上。
2. 上传程序并接通外部 5V，两个舵机会进入 `90` 度。
3. 不要手拧舵机输出轴；在保持 90 度的位置将舵盘装到云台的机械中位，再固定螺丝。
4. 当前安装标定为 Pan `0..180` 度、Tilt `90..162` 度，默认位置为 Pan `93`、Tilt `162`。先分别小幅测试：

```text
pan 80
pan 100
tilt 100
tilt 150
center
```

5. 需要完整动作时发送 `sweep`。Pan 会依次运动到 0、180 和 93 度；Tilt 会依次运动到 90、162 和 162 度。测试时务必观察线材和支架，出现碰撞趋势立即发送 `stop`。

## 串口命令

| 命令 | 作用 |
| --- | --- |
| `pan ANGLE` | 设定水平舵机角度，范围为 0 到 180。 |
| `tilt ANGLE` | 设定竖直舵机角度，范围为 90 到 162。 |
| `set PAN TILT` | 同时设定两个角度。 |
| `center` | 两轴回到 90 度中位，并取消自动巡检。 |
| `sweep` | 自动执行 Pan 0 度、Pan 180 度、回中、Tilt 90 度、Tilt 162 度、回中。 |
| `stop` | 立即取消正在进行的自动巡检，保持当前位置。 |
| `release` | 取消巡检并停止舵机 PWM 保持力。 |
| `attach` | 重新启用已 `release` 的舵机。 |
| `status` | 查看当前角度、限位和自动巡检状态。 |
| `help` | 显示命令帮助。 |

`pan` 角度增大对应物理左还是右、`tilt` 角度增大对应物理上还是下，取决于你安装舵盘的方向。自动 `sweep` 无论方向定义如何都会完整覆盖两个方向；以实际观察到的物理方向为准。

## 自动巡检演示

默认 `AUTORUN_SWEEP = false`，原因是首次安装时不能让云台自动撞向未知的机械边界。确认机构可承受完整行程后：

1. 将 `PAN_CENTER`、`TILT_CENTER` 改为标定好的默认位置。
2. 将 `AUTORUN_SWEEP` 改为 `true`。
3. 重新上传。

之后程序启动时会自动完成左右、上下和回中巡检。巡检是非阻塞的，串口输入 `stop` 或 `center` 可以中断，不会因为 `delay()` 而卡住。

## 与正式小车固件的关系

当前 [car_ctrl_esp32_pwm.ino](../car_ctrl_esp32_pwm/car_ctrl_esp32_pwm.ino) 已经在 `GPIO25` 和 `GPIO26` 初始化了两个舵机，Atlas 下发数据中的两个舵机角度字段依次对应这两个引脚。云台标定完成后，可将正式固件的初始角度更新为实测的中位角度；本调试工程不会自动修改正式固件。

## 本机测试

```powershell
python -m unittest discover -s tests -v
```

该测试验证角度限位和自动巡检顺序，不替代实际接线后的硬件测试。
